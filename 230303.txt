FAR(False Acceptance Rate) 타인 수락률
바이오 인식 시스템의 정확성 비교에 사용되는 기준. 비등록자를 등록자로 잘못 인식할 확률.



음성인식 (STT) 정확도 판별 지표
-WER(Word Error Rate)
D: 음성인식된 텍스트에서 잘못 삭제된 단어 수
S: 음성인식된 텍스트에서 잘못 대체된 단어 수
I: 음성인식된 텍스트에서 잘못 추가된 단어 수
N: 정답 텍스트의 단어 수
WER(단어 에러 비율) = (S + D + I) / N
잘못 인식한 단어 수 / 정답 단어 수 라고 보면 편할 것 같다.

-CER(Character Error Rate)
D: 음성인식된 텍스트에서 잘못 삭제된 음절 수
S: 음성인식된 텍스트에서 잘못 대체된 음절 수
I: 음성인식된 텍스트에서 잘못 추가된 음절 수
N: 정답 텍스트의 단어 수
CER(음절 에러 비율) = (S + D + I) / N
음절의 수를 기준으로 평가하는 것

브로커(broker)
서버에 연결된 구독자(subscribe) 클라이언트와 발행자(publish) 클라이언트 간에 메시지를 전송하는 역할을 함.
게시자로부터 수신되는 모든 메시지를 필터링하고 구독자에게 직접 배포함.

클라이언트(client)
브로커에 연결해 정보를 주고받는 장치.
발행자 클라이언트: 메시지를 전송
구독자 클라이언트: 메시지를 수신

폴링
하나의 장치(또는 프로그램)가 충돌 회피 또는 동기화 처리 등을 목적으로 다른 장치(또는 프로그램)의 상태를 주기적으로 검사하여 일정한 조건을 만족할 때 송수신 등의 자료처리를 하는 방식을 말한다.
실시간처럼 보이는 웹 구현을 위해 폴링을 한다. 폴링의 주기가 짧으면 실시간처럼 보이나 서버 성능에 부담이 감. 주기가 길면 실시간성이 떨어짐 


MQTT
https://webnautes.tistory.com/1644
https://aws.amazon.com/ko/what-is/mqtt/
머신 대 머신 통신에 사용되는 표준 기반 메시징 프로토콜 또는 규칙 세트..
스마트 센서, 웨어러블, 기타 IoT 디바이스는 일반적으로 리소스 제약이 있는 네트워크를 통해 제한된 대역폭으로 데이터를 전송하고 수신해야 함.
MQTT 프로토콜은 게시/구독 모델의 원칙을 기반으로 작동.전통적 네트워크 통신에서는 클라이언트와 서버가 직접 통신한다. MQTT는 게시/구독 패턴을 사용해 발신자와 수신자를 분리한다. 제3의 구성요소인 메시지 브로커가 게시자와 구독자 간의 통신을 처리한다.
(공간분리) 게시자와 구독자는 서로의 네트워크 위치를 모르며 IP주소나 포트 번호와 같은 정보를 교환하지 않는다.
(시간분리) 게시자와 구독자는 동시에 실행되거나 네트워크를 통해 연결되지 않는다.
(동기화 분리) 게시자와 구독자는 서로를 중단시키지 않고 메시지를 전송, 수신할 수 있다. 즉 구독자는 게시자가 메시지를 전송할 때까지 기다리지 않아도 된다.
SSL 포로토콜을 사용해 IoT 디바이스에서 전송되는 민감한 데이터를 보호한다.
